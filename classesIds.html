Classes & Ids

Classes and Ids are very similar, in that they are classification that the writer of the code gives to chunks of code. Classes are more general purpose, as they can be assigned to as many different elements in a page. They are also useful in that they overrule html level selectors. If you have a specific paragraph that you'd like to have red text instead of black, then using a class may be your best choice. When styling based on class, you should use a period prior to the class name. In your stylesheet, this will look like what's on the right.


stylesheet formatting 


      .className{
        color:red;
      }

What seperates classes and Ids is that an Id is a more specific selector. Unlike a class, Ids can only be used once per page, and overrule html selectors as well as classes. Generally, it is best to avoid Ids unless you have a very specific purpose. We use a pound sign/hashtag to style an Id within a stylesheet.



      #idName{
        color:red;
      }
